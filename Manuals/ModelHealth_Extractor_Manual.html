คู่มือการใช้งาน Model Health Extractor

เครื่องมือวิเคราะห์สุขภาพโมเดล Revit และส่งออกแบบอัตโนมัติลง Excel ผ่าน Dynamo

1. ภาพรวมการทำงาน (Overview)

สคริปต์ Dynamo Python Node สำหรับสแกนเจาะลึกโมเดล Revit แบบ 360 องศา ครอบคลุมตั้งแต่ Warnings, Families, Groups, CAD Links ไปจนถึงข้อมูลเบื้องลึกที่หาดูยาก พร้อมระบบ "กันตาย (Bulletproof)" ที่หลบเลี่ยง Bug ของ Revit API และส่งออกข้อมูลลงไฟล์ Excel อย่างเป็นระเบียบด้วยระบบ Outline Grouping

2. สิ่งที่ต้องเตรียมพร้อม (Prerequisites)

สำคัญ การติดตั้ง openpyxl สำหรับ Dynamo

เนื่องจากโค้ดชุดนี้มีการเขียนไฟล์ .xlsx โดยตรงแบบไม่พึ่งพาโหนดของ Dynamo พื้นฐาน (เพื่อหลีกเลี่ยงบั๊กการรวมเซลล์) ผู้ใช้งานจึงต้องมีไลบรารี openpyxl ติดตั้งอยู่ใน Python Environment ของ Dynamo

  • สำหรับ Dynamo 2.13+ (CPython3): ให้ทำการ pip install openpyxl ในโฟลเดอร์ Python ของ Dynamo
  • หรือคัดลอกโฟลเดอร์ openpyxl ไปวางไว้ในพาธที่ Dynamo เรียกใช้งาน (เช่น C:\Users\Username\AppData\Roaming\Dynamo\Dynamo Revit\2.xx\packages)

3. วิธีการนำไปใช้งาน (How to Use)

ขั้นตอนการตั้งค่า Node ใน Dynamo

คัดลอกโค้ด dynamo_extractor_node_final.py ไปวางใน Python Script โหนด โดยตั้งค่าช่องเสียบสายสัญญาณ (Input Ports) ดังนี้:

Input Port ประเภทข้อมูล (Type) หน้าที่
IN[0]Boolean (True/False)สวิตช์เปิด/ปิดการทำงานของ Script (ตั้งเป็น True เพื่อให้ทำงาน)

* เมื่อรัน Script จะมีหน้าต่าง Save File Dialog เด้งขึ้นมาให้คุณเลือกตำแหน่งและตั้งชื่อไฟล์ Excel ที่จะบันทึก หากเลือกไฟล์ที่มีอยู่เดิม โปรแกรมจะทำการ เขียนต่อท้าย (Append) เพื่อเก็บประวัติเปรียบเทียบข้อมูลย้อนหลัง

ข้อควรระวัง: แนะนำให้ตั้งค่าการรัน Dynamo เป็น "Manual" เสมอ เพื่อป้องกันไม่ให้หน้าต่าง Save File เด้งขึ้นมารบกวนตลอดเวลา

4. ทำความเข้าใจไฟล์รายงาน Excel (Output Explanation)

รายงานจะถูกแบ่งออกเป็นชีต (Sheet) ต่างๆ ตามหมวดหมู่ข้อมูล ดังนี้:

ระบบจัดกลุ่มอัจฉริยะ (Outline Grouping)

ในชีตที่มีข้อมูลเยอะ (เช่น Families, Groups, CAD Links) สคริปต์จะไม่ใช้การผสานเซลล์ (Merge Cells) เพราะจะทำให้การทำ Filter ใน Excel พัง แต่จะใช้ฟีเจอร์ Outline Group (+/-) ของ Excel แทน

  • หมวดหมู่หลัก (Level 1): ระบุรอบวันที่การอัปเดต (คลิกปุ่ม - ที่ขอบซ้ายเพื่อพับเก็บข้อมูลทั้งหมด)
  • หมวดหมู่ย่อย (Level 2): ระบุกลุ่มของข้อมูล เช่น 📂 หมวดหมู่: Model Groups หรือ 👤 ผู้สร้าง: John Doe
  • แถวข้อมูล (Level 3): ข้อมูลระดับรายตัวของ Element

รายละเอียดแต่ละ Sheet

  • Summary: ภาพรวมสุขภาพโมเดล แสดงขนาดไฟล์ (MB) และจำนวนไอเทมรวมในทุกหมวดหมู่ ใช้สำหรับประเมินความหนักของโมเดลอย่างรวดเร็ว
  • Warnings: รายการแจ้งเตือนทั้งหมด พร้อม Severity (ความรุนแรง) และ Element ID ของตัวต้นเหตุ
  • InPlaceFamilies: ครอบครัวนอกคอก (In-Place) ซึ่งกินทรัพยากรเครื่องสูงมาก จัดกลุ่มตาม Category เพื่อหว่านหาเป้าหมายในการแก้เป็น Loadable Family
  • LoadableFamilies: รวบรวม Family ทั้งหมดในโมเดล พร้อมนับจำนวน Placement Count อย่างแม่นยำ
  • Groups: ข้อมูล Model Group และ Detail Group พร้อมจำนวนการวาง (Placement Count) และจำนวนของข้างใน (Member Count) *ติดตั้งระบบบายพาสแก้บั๊ก `property cannot be read` ของ Revit ไว้แล้ว
  • CADLinks & LineStylesAndPatterns: ตามล่าหาขยะ CAD ที่หลงเหลืออยู่ในโมเดล โดยจัดกลุ่มตาม "ชื่อผู้สร้าง (Creator)" หากพบชื่อแปลกๆ หรือมาจากไฟล์ .dwg ระบบจะขึ้นแท็ก Is Suspicious = Yes ทันที

5. การแก้ปัญหาเบื้องต้น (Troubleshooting)

  • ไม่มีอะไรอัปเดตลง Excel เลย: ให้ตรวจสอบว่าคุณ "ปิด" ไฟล์ Excel ปลายทางก่อนที่จะกดรันสคริปต์ใน Dynamo หรือยัง? (หากเปิดค้างไว้ สคริปต์จะถูกปฏิเสธสิทธิ์ในการเขียนทับไฟล์)
  • ชื่อ Group เขียนว่า Error Reading...: เกิดจากปัญหาลึกในชั้น Python.NET ของ Dynamo สคริปต์มีการอัปเดตให้หลบเลี่ยงปัญหานี้แล้ว หากยังพบ ให้ตรวจสอบว่ามีการอัปเดต Dynamo เวอร์ชันใหม่ล่าสุดหรือไม่
  • เปิด Excel ไม่ได้ (Corrupted File): เกิดจากการใส่เครื่องหมายเท่ากับ (=) นำหน้าข้อความ ทำให้ Excel มองว่าเป็นสูตร (Formula) สคริปต์แก้ไขให้ใช้ --- นำหน้าแทนแล้ว กรุณาอย่าเปลี่ยนกลับในซอร์สโค้ด
